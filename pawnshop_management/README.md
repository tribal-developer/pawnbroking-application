# Pawnshop / Gold-Loan Management System (Django)

An internal staff tool for a jewelry pawn/mortgage shop: register customers,
pledge items against loans, track interest, record repayments, renew loans,
and handle auctions for defaulted loans. Built with Django + SQLite +
ReportLab (for printable PDFs).

This was generated by Claude as a starting point, **not a finished, audited
product** -- read the "Before you go live" section before using it for real
money or real customer data.

## Features

- **Customers** -- registration with KYC (ID proof), photo, address, search
- **Pledge items** -- weight/purity/appraisal tracking, multiple photos per item
- **Loans** -- principal, monthly interest rate (flat or reducing-balance),
  auto-calculated due date and accrued interest
- **Repayments** -- interest-only, partial, or full redemption, each with
  a generated receipt number
- **Renewals** -- settle accrued interest and push the due date out
- **Auctions** -- list loans overdue past a configurable grace period,
  record the sale, and flag any surplus owed back to the customer
- **Daily metal rates** -- track gold/silver rate per gram (used as a manual
  reference when staff appraise an item -- see "Known simplifications" below)
- **PDF documents** -- pledge ticket (issued at loan creation) and payment
  receipts, generated with ReportLab
- **Dashboard** -- active loans, overdue count, total outstanding, today's
  collections, recent activity
- Full Django Admin access to every model for quick edits/bulk actions

## Setup

```bash
# 1. Create and activate a virtual environment
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Apply migrations
python manage.py makemigrations 
python manage.py migrate

# 4. (Optional but recommended) load demo data: a superuser, sample
#    customers, items, and loans -- one current, one overdue
python manage.py seed_demo_data

# 5. Or create your own admin login instead of the demo one
python manage.py createsuperuser

# 6. Run the dev server
python manage.py runserver
```

Visit `http://127.0.0.1:8000/` and log in. The seed command creates
`admin` / `admin123` -- **change this password immediately** if you keep it.

Django Admin is at `/admin/` -- useful for bulk edits or fixing data.

## Typical day-to-day flow

1. **Customers → Register New Customer** (KYC details, photo, ID proof)
2. From the customer page, **+ New Loan** -- enter item details (weight,
   purity, appraised value) and loan terms (principal, monthly interest,
   tenure). This creates the item and loan together and lets you print the
   pledge ticket.
3. As interest accrues, use **Pay Interest** on the loan page to record a
   payment without touching principal, or **Renew** to settle interest and
   push the due date out.
4. When the customer pays everything back, use **Redeem (Full Payment)** --
   this closes the loan and marks the item redeemed.
5. If a loan goes unpaid past the grace period, it shows up under
   **Auctions → Eligible Loans**. Recording an auction closes the loan and
   flags any surplus owed back to the customer.

## Known simplifications / things to adjust before going live

This covers the core workflow end-to-end, but a few things are intentionally
simplified and **should be reviewed against your local pawnbroking
regulations** (many regions cap interest rates, mandate specific record
formats, or require notice periods before auction):

- **Interest calculation** (`loans/models.py`, `months_elapsed()`): charges
  a full month's interest for any partial month. Switch the
  `ROUND_UP_PARTIAL_MONTH_INTEREST` setting or rewrite the function if your
  shop prorates by day instead.
- **Appraised value** is entered manually by staff per item -- it is *not*
  auto-calculated from the daily metal rate yet. Wire `MetalRate.get_current_rate()`
  into `items/forms.py` / some JS on the loan-creation page if you want
  live auto-suggestion as staff type in weight and purity.
- **No SMS/email reminders** for upcoming due dates -- the dashboard
  surfaces overdue loans, but there's no automated notification yet. Adding
  this means: Celery + Redis (or a simple cron + `python manage.py` command)
  plus an SMS/email provider account.
- **Single shop, single currency (₹)** -- hardcoded in templates and PDFs.
  Search for "₹" / "Rs." if you need to change currency.
- **No customer self-service portal** -- this is staff-facing only. (Said
  you didn't need this when we scoped the project, but flagging in case
  that changes.)
- **SQLite** is fine for development/single-shop low-concurrency use. Move
  to PostgreSQL for production (the settings.py has a commented-out block
  ready to swap in).
- **DEBUG = True, SECRET_KEY hardcoded, ALLOWED_HOSTS = ['*']** in
  `settings.py` -- all fine for local testing, all wrong for production.
  Before deploying: set `DEBUG = False`, generate a real secret key and load
  it from an environment variable, and lock down `ALLOWED_HOSTS`.
- **Shop name/address/phone** in `documents/views.py` (`SHOP_NAME`,
  `SHOP_ADDRESS`, `SHOP_PHONE`) are placeholders -- update them.
- No automated tests yet. Given this handles money and legal documents,
  I'd strongly suggest adding tests for the interest calculation and
  repayment/redemption logic before relying on it.

## Extending it

- Add a `notifications` app + Celery for due-date reminders
- Add a customer-facing read-only portal (loan status, amount due) if you
  decide you want one later
- Add a `reports` app for monthly interest income / collections exports
  (CSV or Excel via `openpyxl`)
- Swap ReportLab templates for branded PDFs (logo, letterhead) if needed
